# Constitution Relationship

A constitution is the root scaffold of a human-agent system.

Doctrine is the principles layer inside that scaffold.

```text
constitution = what must remain true while the system changes
doctrine     = how purpose, ethics, knowledge, governance, incentives, and method constrain action
```

Doctrine should not duplicate identity, project, skills, knowledge, or meta contents.

It should define the rules those layers apply.

## Rule

When doctrine and another layer appear to conflict, resolve the conflict by asking:

```text
Is this a governing principle?
Is this an actor fact?
Is this project-specific context?
Is this a reusable procedure?
Is this a knowledge claim?
Is this system upkeep knowledge?
```

Move the content to the layer that owns the answer.
