# Adaptive Systems

## Smallest justified structure

A structure is justified when it preserves evidence, defines authority, changes future action, prevents a recurring mistake, or routes work to source truth.

```text
disposable task      -> existing instructions
one durable concern  -> one standalone layer
coupled concerns     -> smallest matching profile
unusual need         -> exact custom composition
```

Completeness is not a governing objective. Unused structure creates search cost, stale policy, false confidence, and maintenance burden.

## Knowledge

Use ordinary knowledge for deliberate curation. Add living maintenance only when external source changes can invalidate current claims and the system has freshness, verification, publication, budget, and stopping rules.

## Agents and loops

Capability does not create authority. A recurring loop requires durable state outside model context, bounded work selection, independent verification where useful, explicit budgets, evidence, escalation, and a human-controlled stop condition.

## Graphs and retrieval

Canonical sources remain above indexes, embeddings, graph projections, communities, rankings, and generated summaries. Derived systems may accelerate navigation but remain rebuildable and challengeable.

## Adaptation

Add, revise, or retire structure from observed failures and outcomes. Do not protect a layer, profile, skill, graph, or loop from retirement merely because it already exists.
